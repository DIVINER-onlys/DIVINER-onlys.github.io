概要

很多人在vue的官网学习完之后，往往要开始构建第一个项目时不知道到从何下手，本文的目的就是带领你一步一步使用webpack工具构建出完整的vuejs开发项目的目录

注意：因本文介绍比较详细，在讲解过程中有附上详细代码，因此篇幅较大
项目源码：DIVINER-onlys/SEG-Studio-Vue-Webpack-Project-Standard-github

前提

  • 你已经了解vue的基础知识，尝试过使用vue-cli官方脚手架搭建项目。

安装

要使用npm命令就需要先安装node.js, npm是随node.js安装的，所以需要先安装node.js，因为node.js安装比较容易就不在这里演示，
这里给个传送门：node.js安装下载地址：http://nodejs.cn/download/

一般，我们还会安装淘宝镜像cnpm，因为在墙内，有时候使用npm安装会很慢，所以需要。

vue脚手架的安装

上述命令的就是使用npm安装vue-cli这个脚手架，-g就是全局安装，安装完毕就可以使用vue命令了
我们接下来就使用它的命令行工具 vue-cli 来构建项目。

Step1.0 项目搭建

项目初始化

建立可以运行的项目（打包工具：webpack）（指令作用介绍如下）

（1）找一个盘中的一个文件夹，在里面新建vue项目
vue init webpack my-project // 初始化项目
（2）进入my-project
cd my-project
（3）把配置文件的插件安装一遍
npm install // 安装 项目依赖
（4）运行开发环境
npm run dev // 在localhost启动测试服务器
（5）把项目编译成可以上线的静态html css js
npm run build // 生成上线目录（部署）

Vuex 是什么？

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。Vuex 也集成到 Vue 的官方调试工具 devtools extension，提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。

接下来介绍一下路由守卫

路由守卫是什么

正如其名，vue-router 提供的导航守卫主要用来通过跳转或取消的方式守卫导航。有多种机会植入路由导航过程中：全局的, 单个路由独享的, 或者组件级的。

记住参数或查询的改变并不会触发进入/离开的导航守卫。你可以通过观察 $route 对象来应对这些变化，或使用 beforeRouteUpdate 的组件内守卫。
不知道什么是路由守卫的请传送门 路由守卫介绍

每个守卫方法接收三个参数：

  • to: Route: 即将要进入的目标 路由对象
  • from: Route: 当前导航正要离开的路由
  • next: Function: 一定要调用该方法来 resolve 这个钩子。执行效果依赖 next 方法的调用参数。

为什么使用路由守卫

比如我们在首页要跳转到一些需要权限（比如完成登录）的页面时，我们就需要检验用户是否已经完成登录，没有登录则需要进行登录才能进入页面，为了再进入页面前高效的检验，我们就需要用到路由守卫进行拦截检验
不是很清楚路由守卫作用的可以看我的一篇文章关于登录验证在路由守卫处的两种处理方法
